# Granite SDK 표준 패턴 (앱인토스 미니앱)

WebView 기반 앱인토스 미니앱이 토스 네이티브와 상호작용할 때 쓰는 Granite SDK의 검증된 패턴입니다.
실제 API 시그니처와 최신 메서드는 항상 앱인토스 MCP(`ax mcp`)를 통해 공식 문서를 다시 확인하세요. 이 파일은 자주 틀리는 함정과 강제 규칙만 정리합니다.

---

## 1. 백버튼 — 가장 자주 틀리는 부분

### 강제 규칙

앱인토스 미니앱의 백버튼은 **반드시 `graniteEvent.backEvent`로 처리**합니다.

- ✓ `graniteEvent.backEvent`에 핸들러를 등록한다
- ✗ `history.pushState` / `popstate` 이벤트로 백버튼을 가로채지 않는다
- ✗ `window.onbeforeunload`로 우회하지 않는다

### 왜 그런가

토스 네이티브 셸이 자체 네비게이션 스택을 관리합니다. 웹 표준 `pushState/popstate`로 가짜 히스토리를 쌓아 백버튼을 막으려 하면, 네이티브 스택과 어긋나면서 다음 증상이 생깁니다:

- 백버튼을 눌러도 미니앱이 닫히지 않거나, 한 번 더 눌러야 닫힘
- 모달을 닫는 백버튼이 미니앱 전체를 닫아버림
- iOS와 Android에서 동작이 달라짐

### 패턴

```js
// 모달이 열려있을 때만 모달을 닫고, 그 외에는 기본 동작(미니앱 종료) 허용
graniteEvent.backEvent.addListener(() => {
  if (isModalOpen) {
    closeModal();
    return true;  // 이벤트 소비 → 미니앱 종료 막음
  }
  return false;   // 기본 동작 → 미니앱 종료
});
```

실제 메서드명/반환값 스펙은 MCP에서 확인하되, **"커스텀 백 처리는 graniteEvent를 거친다"** 원칙은 변하지 않습니다.

---

## 2. 화면 전환 / 모달

내부 라우팅(SPA 라우터)은 자유롭게 써도 되지만, 다음은 Granite를 거칩니다:

- 외부 URL 열기: 네이티브 브라우저로 띄우려면 Granite의 외부 링크 API
- 다른 앱인토스 미니앱으로 이동
- 토스 네이티브 페이지(송금, 결제 등) 호출

순수 미니앱 내부 화면 전환은 일반 라우터(history API)로 OK. 단 **백버튼 처리는 위 규칙 적용**.

---

## 3. 인증 / 사용자 정보

토스 사용자 정보가 필요하면 Granite의 인증 API로 토큰을 받아 백엔드와 교환합니다.

- ✗ 미니앱 내에서 자체 로그인 폼 만들지 않기 (토스 사용자는 이미 로그인 상태)
- ✓ Granite로 토스 인증 토큰 요청 → 서버에서 검증 → 세션 발급

구체 메서드는 MCP 문서 참조.

---

## 4. 흔한 함정

| 함정 | 증상 | 해결 |
|------|------|------|
| `pushState`로 백버튼 가로채기 | iOS/Android 동작 불일치, 더블 백 필요 | `graniteEvent.backEvent` 사용 |
| `localStorage`에 민감정보 저장 | 검수 반려 | 토큰은 메모리에만, 영속화 필요시 서버 |
| 외부 URL을 `window.open` | 토스 앱 내에서 새 창 안 뜨거나 깨짐 | Granite 외부 링크 API |
| 풀스크린 가정한 레이아웃 | 토스 헤더와 겹침 | 안전 영역(safe area) 고려 |
| 데스크톱 마우스 이벤트만 처리 | 모바일에서 안 먹음 | 터치 이벤트 또는 pointer 이벤트 사용 |

---

## 5. MCP로 확인할 것

이 파일은 원칙만 정리합니다. 실제 구현 시 MCP에서 다음을 확인하세요:

- 정확한 메서드 시그니처와 반환 타입
- 사용 가능한 이벤트 목록
- 권한 요청이 필요한 API (카메라, 위치, 결제 등)
- 콘솔 등록 시 필요한 권한 선언
- 최신 SDK 버전과 deprecated된 API

```bash
# Granite SDK 문서 조회 (MCP가 연결된 상태)
ax mcp ...
```

# 앱인토스 미니앱 디자인 시스템 (로고·썸네일·UI)

이 문서는 앱인토스에 어울리는 로고·썸네일 디자인 토큰과, 미니앱 UI에서 권장되는 디자인 시스템을 정리합니다. 마케팅 에셋(로고·썸네일)은 **typographic-bold** 단일 패턴을 표준으로 사용합니다. 미니앱 UI 는 토스 공식 디자인 시스템(TDS) 위에 얹습니다.

레퍼런스: `~/Desktop/word5/{logo-600x600,thumbnail-1932x828}.png` — 따뜻한 단색 배경, 다크 한글 큰 글자 + 액센트 한 글자, 검정 솔리드 pill, 장식 없음.

---

## 1. typographic-bold 디자인 토큰

심볼·일러스트·컨테이너·스파클·도트·글로우 없음. **타이포그래피와 컬러 대비로만** 시각 인상을 만듭니다.

### (a) 색상 시스템

세 가지 토큰이 모든 컬러를 결정합니다.

| 토큰 | 의미 | 기본값 (word5) |
|------|------|----------------|
| `--bg` | 캔버스 단색 배경 (warm·saturated 권장) | `#FFCD3C` mustard |
| `--text` | 메인 한글 텍스트, pill 배경 | `#1A1A1A` near-black |
| `--accent` | 액센트 글자(앱명 끝 한 글자/숫자), 강조 컬러 | `#EA3C53` rose-red |

규칙:
- `--bg` 와 `--accent` 는 보색·삼각배색 관계 (대비 강하게)
- `--text` 는 `#1A1A1A` 고정 권장 (`#FFFFFF` 변형은 BG 가 매우 어두운 경우에만)
- pill 배경 = `--text`, pill 텍스트 = `--bg` 와 충분 대비되는 화이트 또는 `--bg`
- 모든 페어링은 WCAG AA 4.5:1 대비 통과 필수

### (b) 카테고리별 컬러 페어링

| 카테고리 | `--bg` | `--accent` | 비고 |
|----------|--------|-----------|------|
| 교육·지식 | `#FFCD3C` mustard | `#EA3C53` rose | word5 기본 |
| 금융/자산관리 | `#3182F6` 토스블루 | `#FFE600` electric yellow | text=`#FFFFFF` 권장 (어두운 BG) |
| 건강/운동 | `#22C55E` vivid green | `#1A1A1A` | accent=다크, text=`#FFFFFF` |
| 라이프스타일 | `#FFAB5C` warm peach | `#1F3F8A` deep navy | |
| 소셜 | `#FFD1E0` soft pink | `#1A1A1A` | |
| 생산성/도구 | `#E8E2D4` cream | `#1F3F8A` deep navy | accent=navy, text=`#1A1A1A` |
| 쇼핑/커머스 | `#FFE0E0` blush | `#E53935` strong red | |
| 엔터테인먼트 | `#1A1A1A` charcoal | `#FFE600` electric yellow | text=`#FFFFFF` |
| 여행/교통 | `#9FE3FF` sky cyan | `#1A1A1A` | |

> `--bg` 는 `granite.config.ts` 의 `brand.primaryColor` 와 동일하게 — 미니앱 UI / 로고 / 썸네일 컬러 일관성.

### (c) 타이포그래피

| 역할 | 폰트 패밀리 | weight | size (로고) | size (썸네일) | 비고 |
|------|------------|--------|-------------|---------------|------|
| 메인 한글 앱이름 | Pretendard / Apple SD Gothic Neo / Noto Sans KR | 900 | 180 (2줄) | 180 | `--text` |
| 액센트 글자 (한 글자) | 동일 | 900 | 180 | 480 (좌측 거대) | `--accent` |
| 영문 태그라인 | Pretendard / Inter / Arial | 800 | 44 | 36 | `--text`, letter-spacing 4~6 |
| 부제 (썸네일) | 동일 한글 패밀리 | 600 | — | 64 | `--text` |
| pill 텍스트 | 동일 한글 패밀리 | 700 | — | 32 | `#FFFFFF` (또는 `--bg`) |

### (d) 강제 규칙 (검수 통과)

- 캔버스 (0,0)부터 (W,H)까지 빈틈없이 `--bg` 단색 채움 (둥근 모서리 X, 투명 X)
- alpha=0 영역 없음 (sharp 검증: 모든 픽셀 alpha=255)
- 텍스트 안에 회사명·URL 노출 X
- 그라데이션·그림자·3D 효과 X (typographic-bold 는 **flat 단색만**)
- 실사 사진·복잡한 패턴·일러스트 X
- 장식 요소 (스파클·도트·글로우·라인) X

## 2. 로고 (600×600) 레이아웃 그리드

```
+--------------------------------+
| 60                             | ← top padding
|                                |
|   {앱이름 라인1}                | ← 한글 2~3자, 900, --text
|   {앱이름 라인2}  {액센트}      | ← 한글 2~3자 + 끝에 액센트 글자 1개 (--accent)
|                                |
|         TAG · LINE             | ← 영문 letter-spaced, 800, --text
|                                |
| 60                             | ← bottom padding
+--------------------------------+
```

규칙:
- 한글 앱이름이 4자 이상이면 2줄로 분할 (예: `데일리/단어 5`)
- 마지막 글자(또는 숫자) 1개를 `--accent` 로 칠해 시각 포인트
- 액센트는 정확히 1글자 — 2글자 이상이면 시각 무게 깨짐
- 영문 태그라인 (선택): 1~2 단어, 4~12 글자, letter-spacing 4
- 액센트 글자가 단독으로 강한 의미를 가지면 (예: `5`, `?`, `★`) 그것만으로 충분

## 3. 썸네일 (1932×828) 레이아웃 그리드

```
+------------------------+----------------------------+
| left ~35%              | right ~65%                 |
|                        |                            |
|                        |  {앱이름} (180pt)          |
|     {액센트 글자}      |  {부제 한글} (64pt)        |
|     (480pt, --accent)  |                            |
|                        |  [pill] [pill] [pill]      |
|                        |                            |
|                        |  ENG · SUB · LINE (36pt)   |
+------------------------+----------------------------+
```

규칙:
- 좌측 액센트 글자: 캔버스 높이의 60~70% 차지하는 거대한 1글자 (숫자·기호·한 글자 한글)
- 우측 상단: 한글 앱이름, baseline 좌측 정렬
- 부제: 앱이름 바로 아래, 한 줄 (20자 이내 권장)
- pill: 3개, 검정 솔리드 (`--text`), 흰 글자, 높이 80, 가로 padding 40, radius 40
- 영문 sub: pill 아래, letter-spacing 6, 작게

## 4. 미니앱 UI 디자인 시스템

비게임 미니앱 UI 는 **TDS(Toss Design System)** 를 사용합니다. `create-ait-app` 부트스트랩 시 TDS 옵션을 켜면 자동 포함됩니다.

- **WebView**: `@toss/tds-mobile` 컴포넌트 (Button, Text 등)
- **React Native**: `@toss/tds-react-native`

TDS 미사용은 비게임 미니앱에서 권장하지 않습니다 (검수에서 일관성 문제 지적될 수 있음).

### (a) 공통 내비게이션 바

토스가 자동 제공:
- 좌측: 미니앱 로고 + `displayName` (콘솔 등록 이름)
- 우측: 더보기 메뉴 (공유·신고·권한·홈추가·용량삭제·알림) + X 닫기 버튼
- 옵션:
  - `withBackButton: true` — 뒤로가기 버튼
  - `withHomeButton: true` — 홈(첫 화면) 버튼, 비게임만
  - `initialAccessoryButton` — 더보기 왼쪽 모노톤 아이콘 1개

**자체 헤더에 뒤로가기·홈·로고·종료 버튼을 또 그리면 중복 노출로 검수 반려**.

### (b) Safe Area

토스 헤더와 겹치지 않도록 safe area 적용. 컨테이너 최상단에 `padding-top: env(safe-area-inset-top)` 또는 SDK 의 `useSafeArea()` 훅 사용.

### (c) 색상·간격

- `brand.primaryColor` = §1.a `--bg` 값. 버튼·강조 요소에 자동 적용됨
- 액센트 컬러 (`--accent`) 는 CTA·강조 텍스트·뱃지에만 절제적으로
- TDS spacing scale (4, 8, 12, 16, 20, 24, 32, 40, 48) 사용 권장

### (d) 반응형 폭

- 최소 320px (구형 iPhone SE)
- 최대 428px (iPhone 14 Pro Max)
- 그 사이를 fluid 로 (vw 단위 또는 max-width 컨테이너)

## 5. 디자인 변형 시 우선순위

새 미니앱 디자인을 만들 때:

1. **카테고리 → `--bg`/`--accent` 페어 결정** (§1.b 표). text 는 BG 명도에 맞춰 `#1A1A1A` 또는 `#FFFFFF`
2. **앱 이름 분할 + 액센트 글자 결정**
   - 한글 4자 이상 → 2줄로 분할 (예: `데일리/단어`)
   - 마지막에 액센트 글자 1개 추가 (앱 정체성을 한 글자로: `5`, `?`, `₩`, `↑`, `★`, `토` 등)
   - 또는 앱 이름 자체의 마지막 글자를 액센트 컬러로
3. **영문 태그라인 (선택)** — 로고 하단, 1~2 단어 (예: `DAILY WORD`, `MONEY LEAF`)
4. **부제·pill·sub (썸네일용)** 결정
   - 부제: 한 줄, 20자 이내
   - pill: 핵심 기능 3개 (각 2~5자)
   - sub: 영문 letter-spaced 1~2구절
5. **로고/썸네일 SVG 템플릿 + 변수 치환으로 생성** (`scripts/render-svg-asset.ts`). sharp 가 PNG 변환까지 처리
6. 동일 `--bg`/`--accent`/액센트 글자로 로고·썸네일 일관성 유지
7. SVG 템플릿이 안 맞는 디자인이면 AI 이미지 fallback (asset-specs.md 프롬프트 가이드)

## 6. 디자인 검수 체크리스트

- [ ] 로고 600×600 PNG, 배경이 캔버스 끝까지 `--bg` 단색
- [ ] 썸네일 1932×828 PNG, 배경 동일
- [ ] 모서리 둥글지 않음, 투명 영역 없음, alpha=255
- [ ] 메인 한글 앱이름이 1/3 축소(200×200 미리보기)에서도 읽힘
- [ ] 액센트 글자 1개만 `--accent` 컬러 (2개 이상이면 무게 깨짐)
- [ ] 그라데이션·그림자·일러스트·장식 요소 없음 (flat 단색만)
- [ ] 영문 태그라인 1단어 (회사명·URL X)
- [ ] `--bg` 가 미니앱 UI (`granite.config.ts` `primaryColor`) 와 동일
- [ ] 로고와 썸네일의 토큰 3개가 모두 일관됨
- [ ] 자체 헤더에 추가 로고 X (공통 내비 로고와 중복 방지)

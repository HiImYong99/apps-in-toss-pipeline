# 앱 이미지 규격 및 키워드 가이드

앱인토스 앱의 로고, 썸네일, 그리고 관련 이미지 생성을 위한 기술 규격과 프롬프트 작성 가이드입니다.

---

## 이미지 규격 (Image Specifications)

| 에셋 | 크기 | 파일 형식 | 용도 | 비고 |
|------|------|---------|------|------|
| **로고** | 600×600px | PNG | 앱 아이콘, 앱스토어 표시 | 정사각형, **꽉 찬 배경 필수 (둥근 모서리·투명 X)** |
| **썸네일** | 1932×828px | PNG | 가로형 프로모션 배너, 광고 | 약 2.33:1 (7:3) 가로 형식, **꽉 찬 배경 필수 (둥근 모서리·투명 X)** |

### ⚠️ 강제 규칙: 꽉 찬 사각형 형태

앱 아이콘과 썸네일은 **크롭되지 않으며 배경색이 포함된 꽉 찬 사각형 형태**로 제출해야 한다. 이는 앱인토스 콘솔 등록 시 강제 규칙이다.

- ✗ 모서리 둥글게(`rx`/`ry`) 처리하지 않는다
- ✗ 가장자리 페이드아웃·그라데이션 알파 처리하지 않는다
- ✗ 투명 배경(alpha=0 영역)을 두지 않는다
- ✓ 배경색이 캔버스 (0,0)부터 (W,H)까지 빈틈 없이 채워져 있어야 한다

**왜?** 토스 미니홈이 등록된 이미지를 자체적으로 크롭·둥근 모서리 처리해서 표시한다. 우리가 미리 둥글게 만들면 두 번 잘려 가장자리가 깨지고, 투명 배경이면 토스 자체 배경과 충돌한다.

**구현 가이드**:
- SVG 배경: `<rect width="W" height="H" fill="..."/>` (rx 절대 X)
- 합성 시 base를 opaque PNG로 출력 (transparent 영역 잔존 X)
- sharp 합성 결과 alpha 채널 검증: 모든 픽셀 alpha=255여야 한다

### 로고 (600×600px, PNG)

- **용도**: 앱 아이콘, 앱인토스 마켓플레이스 표시, 공유 시 섬네일, 공통 내비게이션 바 로고
- **특징**: 정사각형, 단색 또는 제한된 색상 팔레트
- **배경**: **꽉 찬 `--bg` 단색** (alpha=255). 투명 배경 금지 — 토스 미니홈이 자체 크롭/둥근 처리 적용함
- **픽셀 밀도**: 1x 기준 (고해상도 디바이스용 자동 스케일)

### 썸네일 (1932×828px, PNG)

- **용도**: 프로모션 배너, SNS 광고, 앱 소개 페이지
- **특징**: 가로 형식 (약 7:3 비율)
- **배경**: 색상 배경 또는 그래디언트 가능
- **텍스트**: 앱 이름 및 부제 포함 가능

## 로고 이미지 프롬프트 가이드 (AI fallback 전용)

> **주의**: 이 섹션은 **SVG 템플릿(typographic-bold) 으로 표현이 불가한 케이스의 AI 이미지 fallback** 가이드입니다. 1차 시도는 항상 `render-svg-asset.ts` (단색 BG + 큰 한글 + 액센트 글자 + flat 디자인). AI fallback 은 일러스트 중심 표현이 꼭 필요할 때만 사용하고, **AI 결과도 단색 배경 + alpha=255 + 둥근 모서리 없음** 강제 규칙은 동일하게 적용합니다.

로고를 생성하기 위해 AI 이미지 생성 도구(예: DALL-E, Midjourney)에 입력할 프롬프트 작성 방법입니다.

### 추천 스타일

- **디자인 방식**: 미니멀리즘 (minimalism)
- **색상**: 단색 또는 2~3가지 주요 색상
- **배경**: 단색 배경 (흰색, 검은색, 또는 브랜드 컬러)
- **복잡도**: 심플한 심볼 또는 아이콘
- **영감**: 앱스토어 아이콘 스타일 (iOS/Android)
- **꽉 찬 구도**: 주요 심볼이 캔버스의 **80% 이상**을 차지해야 함. 작게 그려놓고 빈 배경이 넓으면 앱 아이콘에서 식별이 안 됨. 프롬프트에 "fills 80% of the canvas, large centered symbol, minimal padding" 명시

### 피해야 할 것

- 실사 사진이나 높은 수준의 사진 리얼리즘
- 복잡한 배경, 패턴, 그라데이션
- 너무 많은 텍스트 (회사명, URL 등)
- 미세한 세부사항 (앱 아이콘으로는 보이지 않음)
- 3D 효과나 그림자 과다 사용
- 심볼이 캔버스 중앙에 **작게** 그려진 구도 (여백 과다)

### 프롬프트 예시 구조

```
[앱 이름] 앱 아이콘, [컨셉 키워드 1], [컨셉 키워드 2],
미니멀 디자인, 단색 배경, 심플한 심볼,
fills 80% of the canvas, large centered symbol, minimal padding,
앱 스토어 아이콘 스타일, 600x600px
```

### 구체적인 프롬프트 예시

#### 예시 1: 영수증 정리 가계부

```
Receipt Reader 앱 아이콘, 영수증과 체크마크,
미니멀 디자인, 흰색 배경, 심플한 심볼,
파란색과 초록색 포인트, 앱 스토어 아이콘 스타일, 600x600px
```

#### 예시 2: 이웃 커뮤니티

```
Neighbor 앱 아이콘, 집과 손 잡기,
미니멀 디자인, 크림색 배경, 따뜻한 톤,
심플한 심볼, 앱 스토어 아이콘 스타일, 600x600px
```

#### 예시 3: 영어 학습

```
Daily English 앱 아이콘, 책과 별,
미니멀 디자인, 연보라색 배경, 심플한 심볼,
노란색 포인트, 앱 스토어 아이콘 스타일, 600x600px
```

### 프롬프트 작성 팁

1. **[앱 이름]**: 실제 앱 이름 명시
2. **[컨셉 키워드]**: 앱의 핵심 기능이나 특징을 2~3개 단어로 표현
   - 가계부 → "영수증", "돈", "차트"
   - 커뮤니티 → "집", "손", "네트워크"
   - 학습 앱 → "책", "펜", "별"
3. **스타일 명령**: "미니멀", "심플", "앱스토어 스타일" 등으로 명확히
4. **색상 지정**: 배경색과 포인트 색상 명시 (3~4가지 이내)
5. **크기**: 마지막에 "600x600px" 명시

## 썸네일 이미지 프롬프트 가이드

프로모션 배너용 썸네일을 생성하기 위한 프롬프트 작성 방법입니다.

### 추천 스타일

- **레이아웃**: 가로형 배너 (약 7:3)
- **구성**: 앱 로고 + 앱 이름 + 부제 (또는 핵심 기능) + 앱 화면 스크린샷 느낌
- **배경**: 그라데이션, 패턴, 또는 테마 색상
- **텍스트 배치**: 좌측 또는 우측에 배치하고 여백 충분히 확보
- **톤**: 밝고 긍정적이며 앱인토스(토스) 미니앱 브랜드 느낌
- **꽉 찬 구도**: 주요 그래픽(심볼/일러스트레이션)이 배너 면적의 **80% 이상**을 차지해야 함. 배너 한 켠에 작게 그리지 않고, 화면을 가득 메우는 임팩트 있는 비주얼로. 프롬프트에 "main graphic fills 80% of the banner, bold and large, edge-to-edge composition" 명시

### 피해야 할 것

- 세로형 구도 (가로형 배너는 가로형이어야 함)
- 텍스트가 너무 작거나 많음 (읽기 어려움)
- 스크린샷이 너무 상세하거나 복잡함 (축소 시 보이지 않음)
- 톤이 너무 어둡거나 불분명한 색상
- 로고와 앱 이름이 두 개 이상 섞임 (혼동 야기)
- 메인 그래픽이 배너 한쪽 구석에 작게 들어간 구도 (여백 과다)

### 프롬프트 예시 구조

```
[앱 이름] 프로모션 배너, [부제 또는 핵심 기능],
가로형 배너, 1932x828px, 약 7:3 비율,
[컨셉 키워드 1], [컨셉 키워드 2],
토스 미니앱 스타일, 밝은 톤, [주요 색상]
```

### 구체적인 프롬프트 예시

#### 예시 1: 영수증 정리 가계부

```
Receipt Reader 프로모션 배너, "영수증 사진만 찍으면 자동 정리",
가로형 배너, 1932x828px, 약 7:3 비율,
영수증, 가계부, 자동화,
토스 미니앱 스타일, 밝고 친근한 톤, 파란색과 초록색
```

#### 예시 2: 이웃 커뮤니티

```
Neighbor 프로모션 배너, "우리 동네 사람들과 연결되는 커뮤니티",
가로형 배너, 1932x828px, 약 7:3 비율,
이웃, 커뮤니티, 공유,
토스 미니앱 스타일, 따뜻하고 포근한 톤, 크림색과 주황색
```

#### 예시 3: 영어 학습

```
Daily English 프로모션 배너, "매일 5분, 영어 단어 5개씩 배우기",
가로형 배너, 1932x828px, 약 7:3 비율,
영어, 학습, 단어,
토스 미니앱 스타일, 밝고 긍정적인 톤, 보라색과 노란색
```

### 프롬프트 작성 팁

1. **[앱 이름]**: 앱의 정확한 이름
2. **[부제]**: 앱의 핵심 이점을 한 문장으로 (20~30자)
3. **[컨셉 키워드]**: 3~4개의 핵심 단어로 앱의 성격 표현
4. **배너 크기**: 정확히 "1932x828px, 약 7:3 비율" 명시
5. **브랜드 톤**: "토스 미니앱 스타일"을 명시하면 일관성 있는 톤 생성
6. **색상 선택**: 앱 로고 색상과 조화되도록 선택 (2~3가지)

## 검색 키워드 구성 가이드

앱인토스 마켓플레이스에서 사용자들이 앱을 찾을 수 있도록 하는 검색 키워드를 10~15개 선정하는 방법입니다.

### 키워드 4가지 구성 요소

#### 1. 카테고리 키워드 (2~3개)

앱이 속한 카테고리의 일반명사를 포함합니다. 사용자들이 카테고리로 검색할 때 매칭됩니다.

**예시**:
- 금융/자산관리: "가계부", "지출관리", "가계"
- 커뮤니티: "커뮤니티", "이웃", "동네"
- 교육: "학습", "영어", "공부"

#### 2. 핵심 기능 키워드 (3~5개)

앱의 가장 중요한 기능이나 특징을 나타내는 키워드입니다.

**예시**:
- 영수증 정리 앱: "영수증", "자동인식", "통계", "분류"
- 커뮤니티 앱: "정보공유", "물물교환", "모임", "이웃"
- 학습 앱: "단어", "발음", "복습", "매일"

#### 3. 사용자 니즈 키워드 (3~5개)

사용자가 가진 문제나 원하는 결과를 표현하는 키워드입니다. 사용자 관점의 검색어입니다.

**예시**:
- 영수증 정리 앱: "간편한가계부", "자동정리", "돈관리", "저축"
- 커뮤니티 앱: "이웃찾기", "정보얻기", "나눔", "동네"
- 학습 앱: "짧은학습", "꾸준히배우기", "쉬운영어", "매일공부"

#### 4. 앱 이름 관련 키워드 (1~2개)

앱의 정확한 이름이나 약자, 또는 비슷하게 인식될 수 있는 단어입니다.

**예시**:
- "Receipt Reader", "영수증리더" (정확한 앱 이름)
- "Neighbor", "네이버" (앱 이름)

### 검색 키워드 작성 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **간결성** | 2~4자 정도의 짧은 단어 또는 2단어 조합 | "가계부" (O) vs "복잡한 가계부 관리" (X) |
| **정확성** | 실제 사용자 검색어를 반영 | "자동인식" (O) vs "인공지능기반처리" (X) |
| **다양성** | 같은 의미의 유사어는 피하기 | "가계부", "지출관리" 동시 사용 가능 |
| **일관성** | 앱의 카테고리와 기능을 반영 | 영어 학습 앱이면 "영어"는 필수 포함 |

### 완성 예시: 영수증 정리 가계부

**카테고리 (2~3개)**:
- 가계부
- 지출관리

**앱 이름 (1~2개)**:

## 이미지 생성 워크플로우 (권장 순서)

이 파이프라인은 **SVG 템플릿 우선, AI 이미지 fallback** 방식입니다. SVG 템플릿은 결정론적이고, **typographic-bold** 단일 패턴 (`design-system.md` §1) — 단색 BG + 큰 한글 + 액센트 글자 1개 + 검정 솔리드 pill — 으로 모든 카테고리를 커버합니다.

### 1단계: 디자인 토큰 결정

`design-system.md` §1.b 카테고리별 페어링 표에서 `--bg` / `--accent` 페어를 정하고, 앱 정체성을 나타내는 **액센트 글자 1개**를 결정합니다.

- `--bg`: 캔버스 단색 배경 (예: word5 `#FFCD3C`)
- `--accent`: 액센트 글자·강조 컬러 (예: word5 `#EA3C53`)
- `--text`: 메인 한글 (기본 `#1A1A1A`, 어두운 BG 면 `#FFFFFF`)
- `titleLine1` / `titleLine2`: 한글 앱이름 (4자 이상이면 2줄 분할)
- `accentChar`: 앱 끝에 붙일 한 글자 (예: `5`, `?`, `★`)

### 2단계: SVG 템플릿 렌더링 (1차 시도)

`scripts/templates/logo-template.svg` 와 `scripts/templates/thumbnail-template.svg` 의 변수 슬롯을 채워 SVG → PNG 변환합니다.

```bash
cd "${CLAUDE_PLUGIN_DATA:-${CLAUDE_PLUGIN_ROOT}}" && npx tsx scripts/render-svg-asset.ts \
  --template logo \
  --bg "#FFCD3C" --accent "#EA3C53" \
  --title-line1 "데일리" --title-line2 "단어" \
  --accent-char "5" \
  --tagline "DAILY WORD" \
  --output "$HOME/Desktop/{app-name}/logo-600x600.png"
```

```bash
cd "${CLAUDE_PLUGIN_DATA:-${CLAUDE_PLUGIN_ROOT}}" && npx tsx scripts/render-svg-asset.ts \
  --template thumbnail \
  --bg "#FFCD3C" --accent "#EA3C53" \
  --accent-char "5" \
  --title-ko "데일리단어" \
  --subtitle "30초 영어 단어 퀴즈" \
  --pills "토익,비즈니스,일상회화" \
  --sub "DAILY WORD · 30 SEC QUIZ" \
  --output "$HOME/Desktop/{app-name}/thumbnail-1932x828.png"
```

### 3단계: 사용자 확인

생성된 PNG 를 사용자에게 보여주고 마음에 드는지 확인. 마음에 들면 4단계 (검수) 로. 안 들면 5단계 (AI fallback).

### 4단계: 최종 검수

- 로고: 앱스토어 아이콘으로 축소했을 때 한글 앱이름 식별 가능
- 썸네일: 1/2 축소에서도 한글 앱이름 읽힘
- 액센트 글자 1개만 `--accent` 컬러로 (2개 이상이면 무게 깨짐)
- alpha=255 (투명 영역 없음), 캔버스 끝까지 `--bg`
- 그라데이션·그림자·일러스트·장식 없음 (flat 단색만)
- `design-system.md` §6 체크리스트 통과

### 5단계: AI 이미지 fallback (필요 시)

SVG 템플릿이 안 맞는 디자인이면 아래 프롬프트 가이드로 AI 이미지 생성을 시도합니다. 결과를 `/tmp/toss-logo-raw.png` 또는 `/tmp/toss-thumbnail-raw.png` 로 저장한 뒤 `scripts/resize-image.ts` 로 600×600 / 1932×828 PNG 로 변환합니다.

(아래 "로고 이미지 프롬프트 가이드" / "썸네일 이미지 프롬프트 가이드" 참조)

# 앱인토스 검수 정책 (출시 통과 기준)

앱인토스 미니앱이 검수를 통과하려면 토스가 공개한 다음 정책을 모두 만족해야 합니다. 이 문서는 공식 출처를 요약한 운영 가이드이며, 최신 정책은 항상 `ax search docs --query "<주제>"` 로 재확인하세요.

## 1. 출시 불가 카테고리·콘텐츠 (서비스 오픈 정책)

다음에 해당하면 **검수 통과 자체가 불가능**합니다. Phase 1 브레인스토밍 단계에서 자동 배제하세요.

1. **디지털 자산·가상자산** — NFT 포함, 소유·이전·저장·거래·중개·발행 기능 전부
2. **자금세탁 가능성** — 현금/유사 자산의 직접 교환·전환·환불 흐름
3. **불법·부정행위 조장** — 신분 조작, 해킹, 불법 문서 제공, 우회 정보 수집
4. **사행성·복권·베팅성 콘텐츠** — 재산 손실·중독 우려가 있는 모든 게임성 보상 흐름
5. **금융 상품 중개·판매·광고** — 대출, 보험, 카드, 증권 (인허가 여부와 무관)
6. **투자 자문·리딩방·유료 정보 제공** — 종목 추천, 투자 전략 안내
7. **의료 행위·병원 마케팅** — 비대면 진료, 의료 행위 연결, 병원 예약·광고·홍보 수익 구조
   - 예외: 공공데이터 기반 의료 정보 조회 (순위·추천·예약·광고비 없음, 출처 명시)
8. **단순 자사 홍보** — 기존 회사·서비스를 알리기만 하는 미니앱

추가 제한:
- **쇼핑몰** — 고객센터·가품 방지·환불 정책 없으면 불가
- **교육** — 국가 자격증 없이 수익이 발생하는 서비스 불가

## 2. 어뷰징 방지 정책

동일 워크스페이스 내에서 다음은 등록·운영 제한 사유입니다.

- **유사 핵심 기능 중복 출시 금지** — 주제·테마만 다른 사실상 같은 앱 (예: "AI 여자 얼굴 만들기" + "AI 남자 얼굴 만들기")
- **동일 브랜드 다중 출시 금지** — 이름·로고·색상이 동일한 미니앱을 여러 개 등록
- 예외: 같은 브랜드라도 **서비스 목적과 핵심 기능이 명확히 다르면** 각각 출시 가능 (예: "A 렌탈" + "A 청소")

→ 새 아이디어가 기존 출시 앱과 핵심 기능이 같다면 기존 앱 업데이트로 처리.

## 3. 다크패턴 방지 (UX 최소 기준)

다음 5가지 패턴은 비게임 미니앱 검수에서 즉시 반려됩니다.

1. **진입 직후 바텀시트 금지** — 알림 동의 요청 포함. 사용자가 첫 화면을 보기 전에 전면 차단 X
2. **뒤로가기 시 바텀시트 금지** — 이탈을 막기 위한 알림 동의·재시도 유도 X
3. **나갈 선택지 없는 강제 CTA 금지** — "확인" 하나만 있고 거절·뒤로갈 수 없는 구조 X
4. **예상 못한 시점의 광고 금지** — 사용자 행위와 무관한 전면 광고, 리워드 광고 진입 전 갑작스러운 노출 X
5. **모호한 CTA 라벨 금지** — 버튼만 보고 다음 행동을 예측 못 하는 라벨, CTA 위 중복 보조 설명

## 4. 자사 앱 설치 / 외부 링크 제한

앱인토스의 핵심 가치 = **앱 설치 없이 토스 안에서 끝나는 경험**.

### (a) 자사 앱 설치 유도 전면 금지

- "앱을 설치하시면 더 많은 기능을 이용할 수 있어요" 류 문구 X
- 앱 마켓 링크 삽입 X
- 자사 앱 설치 시 혜택 제공 안내 X
- 설치 유도 배너·이미지 삽입 X

### (b) 외부 링크는 제한적 허용

원칙: **콘솔에 등록한 '앱 내 기능'은 미니앱 내에서 완결**되어야 합니다.

금지 예시:
- 외부 결제창으로 이동
- 외부 회원가입 페이지로 이동
- 공유 링크가 자사 웹사이트로 랜딩
- 주요 기능이 외부 링크에 의존

허용 예시:
- 법률상 고지·필수 안내
- 공공기관·제휴기관 공식 페이지
- 단순 정보 확인용 타사 웹사이트 이동
- 미니앱 기능 후 최저가 구매 플랫폼 / 쿠폰 발급 플랫폼 이동

외부 링크를 띄울 때는 Granite 외부 링크 API를 거치고, `window.open` 직접 호출은 금지 (granite-sdk.md 참조).

## 5. 생성형 AI 서비스 의무

AI로 텍스트·이미지·음성·영상을 생성·요약·추천하는 미니앱은 반드시:

### (a) 사전 고지

- 사용자가 서비스를 처음 이용하거나 AI 기능을 최초로 사용하는 시점에 **"이 기능은 생성형 AI를 활용해요"** 안내 화면을 노출.

### (b) AI 결과물 표시 의무

- AI가 만든 결과물에 **라벨·배지·워터마크 등 사람이 즉시 인식 가능한 방식**으로 표시.

위반 시 최대 3,000만 원 과태료, 서비스 중지·시정명령 대상.

## 6. 광고 운영 정책 (실무 반려 빈발 항목)

광고를 사용하는 미니앱은 다음을 모두 만족해야 합니다.

### (a) 일시 화면 배너 광고 금지

스플래시·로딩·광고 SDK 로드 대기·결과 토스트·짧은 전환 애니메이션 등 **잠깐 떴다 사라지는 화면에 배너 광고 금지**. 배너는 상시 노출되는 메인/서브 화면에만 배치.

### (b) 목업·플레이스홀더 광고 금지

리워드/전면 광고 SDK 로드 중 빈 영역을 채우는 가짜 광고 이미지·"샘플 광고"·테스트 광고 그래픽 노출 금지. 광고 SDK가 실제 광고를 받아온 뒤에만 표시.

### (c) 광고 노출 사전 인지 UI

사용자가 광고를 보게 될 것을 미리 알 수 있도록:
- 리워드 광고 → "광고 보고 보상 받기" 같이 광고 시청임이 명시되는 버튼 라벨, AD 뱃지
- 배너 광고 → 광고 영역 안 또는 가까이에 "광고"/"AD"/"Sponsored" 라벨, 콘텐츠와 시각적 분리
- 전면 광고 → 전환 직전 "잠시 후 광고가 표시됩니다" 동등 안내

### (d) AdMob 정책 준수

토스 인앱 광고 SDK는 내부적으로 Google AdMob을 사용합니다. AdMob 정책 위반(클릭 유도, 광고 옆 의도적 클릭 트랩 등)도 그대로 반려 사유.

광고 API 시그니처(loadFullScreenAd/showFullScreenAd, TossAds.initialize/attachBanner)는 `ax search docs --query "광고"` 로 최신 확인.

## 7. 기술 제약 (Granite SDK)

- **iframe 사용 금지** — 단, YouTube 영상 콘텐츠 embed 는 예외
- **자체 로그인 폼 금지** — Granite 인증 토큰 사용
- **`localStorage` 민감정보 저장 금지** — 토큰은 메모리, 영속화는 서버
- **자체 종료 버튼/헤더 뒤로가기 버튼 금지** — 토스 공통 내비게이션 바가 제공
- **pinch-zoom 비활성화** — viewport meta `user-scalable=no, maximum-scale=1.0`
- **외부 URL 열기** — Granite 외부 링크 API 사용, `window.open` 금지

자세한 패턴은 `granite-sdk.md` 참조.

## 8. 외부 마케팅·홍보 (운영 단계)

출시 후 외부 채널에서 미니앱을 홍보할 때:

- 표기는 **"토스 미니앱"** 또는 **"앱인토스"**
- 금지 표현: "토스 공식 파트너사", "토스 입점사", "토스 제휴사"
- 토스 로고 단독 사용 금지, 토스 UI/아이콘 사용 금지
- 앱인토스 로고 사용 시 채널톡 사전 확인
- 토스 미니앱 배지·마케팅 디자인 템플릿은 공식 제공분 사용

## 9. 검수 체크리스트 핵심 요약

Phase 4 에서 다음을 모두 확인:

**카테고리·정책**
- [ ] 출시 불가 카테고리(§1) 해당 없음
- [ ] 어뷰징 방지(§2) 위반 없음 — 동일 워크스페이스 유사앱·동일 브랜드 다중 출시 X
- [ ] 자사 앱 설치 유도(§4a) 없음
- [ ] 외부 링크가 핵심 기능을 대체하지 않음(§4b)
- [ ] 생성형 AI 사용 시 사전 고지 + 결과물 표시(§5)

**UX**
- [ ] 진입 직후 바텀시트(광고/알림 동의) 없음
- [ ] 뒤로가기 시 바텀시트 없음
- [ ] 모든 화면에 나갈 선택지 존재
- [ ] CTA 라벨이 다음 행동을 명확히 표현

**광고 (해당 시)**
- [ ] 일시 화면에 배너 광고 없음
- [ ] 광고 SDK 로드 중 가짜·목업 광고 없음
- [ ] 광고 노출 사전 안내 UI 존재 (AD 뱃지·라벨)
- [ ] 예상 못한 시점 광고 노출 없음

**기술**
- [ ] iframe 미사용 (YouTube 예외)
- [ ] viewport meta `user-scalable=no`
- [ ] 자체 헤더 뒤로가기·로고·종료 버튼 없음 (공통 내비가 담당)
- [ ] 자체 로그인 폼 없음
- [ ] `localStorage`에 토큰·결제정보 없음
- [ ] 외부 URL 호출은 Granite 외부 링크 API 경유

**기능 완결성**
- [ ] 콘솔에 등록한 '앱 내 기능'이 미니앱 안에서 완결됨
- [ ] 외부 결제창·외부 회원가입으로 빠지는 흐름 없음

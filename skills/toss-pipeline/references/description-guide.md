# 앱 상세 설명 작성 가이드

앱의 가치와 사용 경험을 명확하게 전달하는 설명을 작성하기 위한 가이드입니다. AI 기반 마케팅 자동 생성에 최적화되어 있습니다.

---

## 구조: 5가지 필수 섹션

### 1. [한 줄 소개] — 서비스가 뭔지 바로 알 수 있는 문장

**작성 목표**: 누군가 이 앱을 3초 안에 이해할 수 있도록 작성

**원칙**:
- 앱 이름 또는 서비스명 포함
- 주요 기능 또는 혜택 1개만 명시
- 15자~30자 정도의 짧은 문장

**예시**:
- ✓ "영수증 사진만 찍으면 자동으로 정리해주는 가계부"
- ✓ "매일 5분 투자로 영어 단어를 외우는 앱"
- ✓ "내 동네 이웃들과 정보를 나누는 커뮤니티"
- ✗ "첨단 AI 기술로 금융 라이프스타일을 최적화" (추상적)

**분량**: 1문장, 20~40자

---

### 2. [해결하는 문제] — 왜 필요한지

**작성 목표**: 사용자가 이 앱을 왜 필요로 하는지 공감 유도

**원칙**:
- 실제 불편함이나 고민을 언어화
- 감정적 공감 또는 시간/돈 손실을 표현
- 2~3가지 주요 문제점만 선택

**예시**:
- ✓ "매달 영수증 정리하는데 30분 이상 걸려요. 혹시 빠진 게 있는 건 아닐까 걱정되고요."
- ✓ "영어 공부를 시작하고 싶은데 복잡한 커리큘럼이나 비싼 학비가 부담돼요."
- ✓ "옆집에 누가 사는지 모르고, 내가 필요한 정보를 물어볼 사람이 없어요."
- ✗ "현대인의 디지털 리터러시 부족" (너무 추상적)

**분량**: 2~4문장, 150~250자

---

### 3. [사용 흐름] — 사용자가 보고, 누르고, 경험하는 것을 순서대로

**작성 목표**: 실제 사용 경험을 스토리로 전달하여 구체성 높이기

**원칙**:
- 사용자 시점(일인칭)으로 작성
- "열다" → "누르다" → "입력하다" → "본다" 등 구체적 동사 사용
- 화면 흐름과 결과를 명확히 표현
- 최초 접근부터 목표 달성까지의 시나리오

**예시**:
```
앱을 열면 큼직한 "영수증 촬영" 버튼이 보여요.
버튼을 누르고 영수증을 사진으로 찍으면,
AI가 가게명, 카테고리, 금액을 자동으로 인식해줘요.
잠깐, 뭔가 잘못됐나? 싶으면 한 번 눌러서 수정하기만 해도 돼요.
저장을 누르면 자동으로 가계부에 기록되고,
매월 통계 화면에서 "음식비가 지난달보다 5만원 줄었네!" 하고 확인할 수 있어요.
```

**분량**: 4~6문장, 250~400자

---

### 4. [핵심 기능 3~5개] — 불릿 포인트

**작성 목표**: 앱을 차별화하는 주요 기능을 명확하게 나열

**원칙**:
- 각 기능을 한 줄에, 12~25자 정도로 간결하게
- "자동 인식", "실시간 추적" 같은 기술 용어보다 "사진만 찍으면 된다", "한눈에 본다" 같은 사용자 혜택으로 표현
- 사용자 입장에서 가장 중요한 순서대로 배열
- 기능별로 간단한 설명 덧붙이기 (괄호 안, 20자 이내)

**예시**:
```
• 영수증 자동 인식 (사진만 찍으면 AI가 자동 분류)
• 카테고리별 통계 (월별/주별 지출 현황을 그래프로 한눈에)
• 반복 지출 알림 (같은 금액 결제 시 패턴 인식해서 조언)
• 가족 공유 기능 (가계부를 가족과 함께 관리)
• 저축 목표 추적 (목표액까지 얼마나 남았는지 시각화)
```

**분량**: 3~5개 항목, 총 200~300자

---

### 5. [이런 분께 추천해요] — 타겟 사용자 명시

**작성 목표**: 어떤 사람이 이 앱을 사용하면 가장 만족할지 명확히 하기

**원칙**:
- 구체적인 사용자 페르소나 3~4개
- "누구" + "왜" 형태로 작성
- 자신의 상황과 일치한다고 느낄 수 있도록 표현

**예시**:
```
• 매달 영수증을 정리하느라 시간을 낭비하는 직장인
• 월급의 얼마나 쓰는지, 어디에 쓰는지 한번에 알고 싶은 분
• 가족 생활비를 함께 관리하고 싶은 부부
• 저축 목표를 세웠지만 자꾸 실패하는 분
```

**분량**: 3~4개 항목, 총 150~250자

---

## 작성 원칙 (전체 적용)

| 원칙 | 설명 | 예시 |
|------|------|------|
| **쉬운 단어** | 전문 용어나 외래어 최소화 | "인증" (O) vs "오썬티케이션" (X) |
| **구체성** | 추상적 표현 피하기 | "시간 절약" (X) vs "30분 단축" (O) |
| **사용자 시점** | "저는", "나는" 중심으로 작성 | "사용자가 할 수 있다" (X) vs "당신이 할 수 있어요" (O) |
| **현재형/진행형** | 살아있는 경험처럼 표현 | "할 수 있습니다" (X) vs "해요" (O) |
| **이점 강조** | 기능보다 이점 먼저 | "자동 분류 알고리즘" (X) vs "사진만 찍으면 자동으로 정리" (O) |
| **수치 활용** | 정량적 표현으로 설득력 높이기 | "빠르다" (X) vs "3초 안에 완료" (O) |

## 완성 예시: "영수증 정리 가계부"

### 한 줄 소개
영수증 사진만 찍으면 자동으로 정리해주는 똑똑한 가계부

### 해결하는 문제
매달 받은 영수증을 일일이 정리하다 보면 시간이 한참 걸리고, 혹시 중복으로 입력했거나 빼먹은 게 있는 건 아닐까 걱정돼요. 그래서 많은 사람들이 가계부를 포기하고 맙니다. 하지만 우리는 아무것도 어렵지 않게 만들고 싶었어요.

### 사용 흐름
앱을 열면 빨간 "영수증 촬영" 버튼이 한눈에 들어와요.
버튼을 누르고 영수증을 사진으로 찍으면, AI가 가게 이름, 카테고리, 구매 금액을 자동으로 읽어내요.
혹시 잘못 인식됐다면 한 번만 눌러서 직접 수정하면 되고, 정확하면 그냥 "저장" 버튼을 누르면 끝.
매달 통계 화면에서는 식비, 교통비, 엔터테인먼트별로 얼마를 썼는지 그래프로 확인할 수 있어요.
지난달과 비교해서 어디를 덜 쓰고 어디를 더 썼는지도 한눈에 볼 수 있어요.

### 핵심 기능
• AI 자동 인식 (영수증을 사진으로 찍으면 금액과 품목이 자동 입력)
• 카테고리 자동 분류 (식비, 교통, 쇼핑 등 자동으로 정리)
• 월별 통계 (지출 현황을 그래프로 한눈에 파악)
• 반복 지출 알림 (매달 같은 금액이 나가면 자동으로 알려줌)
• 가족 공유 (가계부를 가족과 함께 관리 가능)

### 이런 분께 추천해요
• 매달 영수증 정리하는데 시간을 낭비하는 직장인
• 월급이 어디로 새는지 확인하고 싶은 분
• 저축 목표를 정해놨지만 자꾸 실패하는 분
• 가족 생활비를 투명하게 관리하고 싶은 부부

## AI 마케팅 자동 생성에 최적화하기

이 가이드를 따라 작성하면 다음과 같은 마케팅 자료가 쉽게 생성됩니다:

- SNS 광고 카피: [한 줄 소개] + [해결하는 문제] 활용
- 앱 스토어 설명: [한 줄 소개] ~ [핵심 기능] 순서대로 배열
- 이메일 뉴스레터: [사용 흐름]을 스토리텔링으로 전개
- 홍보 이미지 텍스트: [타겟 사용자]에 맞춰 메시지 변형

**핵심**: 구체적이고 쉬운 설명일수록 AI가 다양한 마케팅 자료를 자동으로 생성하기 쉬워집니다.

## 작성 양식 (Placeholder Skeleton)

description.md 파일을 작성할 때 그대로 복사해서 placeholder를 채우면 됩니다.

```markdown
# {{앱이름}}

## 한 줄 소개

{{15~40자, 서비스가 뭔지 바로 알 수 있는 문장}}

## 해결하는 문제

{{150~300자. 사용자가 겪는 구체적 불편 1~2개 + 기존 방식의 한계 + 이 앱이 어떻게 해결하는지}}

## 사용 흐름

{{250~500자. 첫 진입부터 목표 달성까지 사용자 시점(일인칭)으로 풀어 쓰기}}

1. {{첫 번째 단계}}
2. {{두 번째 단계}}
3. {{세 번째 단계}}

## 핵심 기능

- {{기능 1: 사용자 혜택 중심 한 줄, 12~25자}}
- {{기능 2}}
- {{기능 3}}
- {{기능 4}}

## 이런 분께 추천해요

{{100~300자. 타겟 사용자 페르소나 3~4개를 "누구 + 왜" 형태로}}
```

placeholder를 채울 때는 위쪽 가이드의 분량 기준과 작성 원칙(쉬운 단어, 구체성, 사용자 시점, 현재형, 이점 강조, 수치 활용)을 그대로 따르면 됩니다.
